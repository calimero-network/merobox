Command Overview

merobox provides seven top-level commands for workflow orchestration, node management, and environment control.

merobox run

Execute a workflow YAML file. This is the primary command — it orchestrates the full lifecycle from environment setup through step execution to cleanup.

merobox run <workflow.yaml> [OPTIONS]

Arguments:
  workflow         Path to workflow YAML file  required

Options:
  --nuke           Destroy existing environment before run
  --no-cleanup     Keep nodes running after workflow completes
  --backend        Force backend: "docker" or "binary" (default: auto)
  --image          Docker image override (e.g. merod:dev)
  --binary-path    Path to merod binary (for binary backend)
  --near-sandbox   Enable NEAR sandbox (auto-detected from YAML)
  --env KEY=VAL     Set environment variable (repeatable)
  --verbose / -v   Increase log verbosity
  --dry-run        Validate workflow without executing

merobox stop

Stop all running nodes managed by merobox.

merobox stop [OPTIONS]

Options:
  --name           Stop specific node by name
  --all            Stop all managed nodes (default)
  --force          Force kill (SIGKILL instead of SIGTERM)
  --timeout        Graceful shutdown timeout in seconds (default: 10)

merobox health

Check the health status of all running nodes.

merobox health [OPTIONS]

Options:
  --name           Check specific node by name
  --json           Output in JSON format
  --wait           Wait until healthy (with timeout)
  --timeout        Wait timeout in seconds (default: 30)

merobox logs

View or follow log output from running nodes.

merobox logs [OPTIONS]

Options:
  --name           Node name to show logs for  required
  --follow / -f    Follow log output (like tail -f)
  --tail           Number of lines from end (default: 100)
  --since          Show logs since timestamp or duration (e.g. "5m")

merobox nuke

Destroy all merobox-managed resources: containers, volumes, binary processes, PID files, and temp data.

merobox nuke [OPTIONS]

Options:
  --confirm / -y   Skip confirmation prompt
  --keep-images    Don't remove pulled Docker images
  --keep-sandbox   Don't remove NEAR sandbox data

merobox remote

Manage remote node registrations via the portal API.

merobox remote <subcommand> [OPTIONS]

Subcommands:
  add              Register a new remote node
    --name         Node name  required
    --url          Node endpoint URL  required
    --token        Auth token (or use $MEROBOX_TOKEN)

  remove           Unregister a remote node
    --name         Node name to remove  required

  list             List all registered remote nodes
    --json         Output in JSON format

merobox bootstrap

Initialize a new merobox project with default configuration files and directory structure.

merobox bootstrap [OPTIONS]

Options:
  --dir            Target directory (default: current)
  --template       Workflow template: "minimal", "e2e", "near" (default: minimal)
  --nodes          Number of nodes to configure (default: 2)
  --with-sandbox   Include NEAR sandbox configuration

Creates the following structure:

./
  workflow.yaml     # Starter workflow from template
  contracts/        # Directory for WASM contracts
  .merobox/         # Local config and cache
    config.toml     # Default node configuration
    tokens/         # Auth token cache

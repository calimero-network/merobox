Top-Level Schema

Every workflow YAML file has three top-level keys.

# workflow.yaml
name: string         # Workflow display name
nodes: NodeConfig    # Node definition (count-based or explicit)
remote_nodes: list  # Optional remote node declarations
steps: list[Step]   # Ordered sequence of workflow steps

Node Configuration

Nodes can be defined using count-based (dynamic) or explicit (named) mode.

Count-Based Mode

Creates count nodes with auto-generated names using the prefix.

nodes:
  count: 3
  prefix: node     # → node-0, node-1, node-2
  ports:
    rpc_start: 2528
    swarm_start: 2428

Explicit Mode

Name each node individually with specific settings.

nodes:
  explicit:
    - name: alice
      rpc_port: 2528
      swarm_port: 2428
    - name: bob
      rpc_port: 2529
      swarm_port: 2429

Remote Nodes

Remote nodes are pre-existing merod instances outside the local environment.

remote_nodes:
  - name: gateway
    url: https://gateway.example.com
    auth_token: ${GATEWAY_TOKEN}
  - name: relay
    url: ${RELAY_URL}

Step Type Reference

Complete field reference for each step type. required fields must be present; all others are optional.

install_application

Installs a WASM application on a target node.

- type: install_application
  name: string              # Step name for results reference  required
  node: string              # Target node name  required
  path: string              # Local path to .wasm file
  url: string               # URL to download .wasm
  metadata: dict            # Application metadata

Output: { "application_id": "..." }

create_context

Creates a new context binding an application to a group on a node.

- type: create_context
  name: string              required
  node: string              required
  application_id: string    # Or ${results.install.application_id}  required
  params: dict              # Context initialization parameters
  seed: string              # Deterministic context ID seed

Output: { "context_id": "...", "member_public_key": "..." }

call

Invokes a method on an application within a context.

- type: call
  name: string              required
  node: string              required
  context_id: string        required
  method: string            # Method name to invoke  required
  args: dict                # Method arguments (JSON serialized)
  expected: any             # Expected return value
  executor_key: string      # Specific key to sign with
  retry: RetryConfig       # Retry settings on failure

Output: { "result": ..., "output": "..." }

wait_for_sync

Waits until all specified nodes have synchronized state for a context.

- type: wait_for_sync
  name: string              required
  context_id: string        required
  nodes: list[string]       # Nodes to check (defaults to all)
  timeout: int              # Timeout in seconds (default: 60)
  poll_interval: float      # Seconds between polls (default: 2.0)
repeat

Loops a sequence of sub-steps a specified number of times.

- type: repeat
  name: string              required
  count: int                # Number of iterations  required
  steps: list[Step]         # Sub-steps to repeat  required
  iter_var: string          # Variable name for current index
parallel

Executes multiple sub-steps concurrently.

- type: parallel
  name: string              required
  steps: list[Step]         # Steps to run concurrently  required
  fail_fast: bool           # Cancel remaining on first failure (default: true)
script

Runs an arbitrary shell command or script.

- type: script
  name: string              required
  command: string           # Shell command to execute  required
  cwd: string               # Working directory
  env: dict                 # Additional environment variables
  timeout: int              # Timeout in seconds

Output: { "exit_code": 0, "stdout": "...", "stderr": "..." }

assert

Validates a condition against workflow results or step output.

- type: assert
  name: string              required
  value: string             # Value to check (supports placeholders)  required
  equals: any               # Expected equality
  contains: string          # Substring match
  matches: string           # Regex pattern
  not_null: bool           # Assert value is not null/empty
fuzzy_test

Randomized testing for eventual consistency verification.

- type: fuzzy_test
  name: string              required
  context_id: string        required
  method: string            required
  iterations: int          # Number of random calls (default: 100)
  arg_generator: string     # Python expression for random args
  sync_after: bool         # Wait for sync between iterations
  verify_consistency: bool  # Check all nodes agree at end

Complete Example

A full workflow demonstrating node setup, application installation, context creation, and method invocation.

name: e2e-counter-test

nodes:
  count: 2
  prefix: node
  ports:
    rpc_start: 2528
    swarm_start: 2428

steps:
  - type: install_application
    name: install_counter
    node: node-0
    path: ${CONTRACT_DIR}/counter.wasm

  - type: create_context
    name: create_ctx
    node: node-0
    application_id: ${results.install_counter.application_id}

  - type: call
    name: increment
    node: node-0
    context_id: ${results.create_ctx.context_id}
    method: increment

  - type: wait_for_sync
    name: sync
    context_id: ${results.create_ctx.context_id}

  - type: call
    name: verify
    node: node-1
    context_id: ${results.create_ctx.context_id}
    method: get_count
    expected: 1
